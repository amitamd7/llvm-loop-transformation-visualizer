# Test cases

**Recommended ÔÇö from C source (fully automated):**

```bash
./run_pipeline.sh testcases/jacobi_stencil.c licm
./run_pipeline.sh testcases/gpu_reduction.c loop-unroll --gpu gfx90a
```

This compiles the C source, applies the LLVM pass, runs `ll-dump` + profiler + remarks, and serves the visualization. All artifacts are generated by the tool's own scripts (including AMDGPU: `--gpu` runs the documented `clang`/ `opt` recipe inside `run_pipeline.sh`, not in an external assistant).

**From pre-existing IR files:**

```bash
./scripts/analyze.sh testcases/loop_tiling_before.ll testcases/loop_tiling_after.ll
```

This runs **`ll-dump`** on both IR files, then **`run-perf.sh`** (CPU) or **`run-rocprof.sh`** (GPU triple / `PROFILER=rocprof` / `GPU_ARCH=`), optionally **`OPT_PASS=...`** for `remarks.json`, and starts the HTTP server.

Optional LLM setup for **Insights** / **Chat** is in **FEATURES.md**.

## CPU test cases

### Jacobi stencil (`jacobi_stencil.c`)

A 2D Jacobi iteration kernel ÔÇö good for demonstrating LICM (loop-invariant code motion), loop-interchange, and other middle-end passes on nested loops.

```bash
./run_pipeline.sh testcases/jacobi_stencil.c licm
./run_pipeline.sh testcases/jacobi_stencil.c loop-interchange
```

### Loop tiling (`loop_tiling_before.ll` / `loop_tiling_after.ll`)

- **Before:** one counted loop over `i`, load/add/store on `A[i]`.
- **After:** same logic with a fixed tile size of **4**: outer induction `t`, inner `i` within `[t, min(t+4, N))`.

Both define `void @tile_demo(ptr %A, i32 %N)` so the viewer's **Show diff** mode can compare the same function name.

```bash
./scripts/analyze.sh testcases/loop_tiling_before.ll testcases/loop_tiling_after.ll
```

Or run:

```bash
testcases/run_loop_tiling_view.sh
```

### Performance: `perf_compare.json`

Whole-function before/after counters for the grouped benchmark chart:

```bash
./scripts/run-perf.sh testcases/loop_tiling_before.ll testcases/loop_tiling_after.ll web/
```

Writes `web/perf_compare.json` with **`runs`** (default 5) ÔÇö the UI labels these as an average over that many executions.

### Optimization remarks

With LLVM **`opt`** on `PATH`:

```bash
OPT_PASS=licm ./scripts/analyze.sh testcases/loop_tiling_before.ll testcases/loop_tiling_after.ll
```

## GPU test cases

### GPU reduction kernel (`gpu_reduction.c`)

A bare `amdgpu_kernel` reduction kernel with a fixed trip count (256 iterations). Use the AMDGPU path so `before.ll` / `after.ll` are produced entirely by the script:

```bash
./run_pipeline.sh testcases/gpu_reduction.c loop-unroll --gpu gfx90a
# or: GPU_MCPU=gfx90a ./run_pipeline.sh testcases/gpu_reduction.c loop-unroll
```

The script runs `clang -target amdgcn-amd-amdhsa -mcpu=ÔÇŽ -O1 -fno-unroll-loops` Ôćĺ `before.ll`, then `opt -passes=<your pass>` Ôćĺ `after.ll` (needed because `-O0` + `mem2reg` on AMDGPU often leaves stack form that loop passes skip).

`rocprof` may fail for bare device IR without a host link harness; `run_pipeline.sh` continues and you still get CFG JSON. For counter data, use OpenMP offload + `build_dataset_amdgpu_loop_unroll.sh` / `rocprof-compare-bins.sh`.

### AMDGPU offload unroll (`amdgpu-loop-unroll/`)

**`bench_offload.c`** ÔÇö OpenMP target loop on GPU. Two Clang builds differ by **`-fno-unroll-loops`** vs **`-funroll-loops`**; LLVM IR is emitted for **`ll-dump`**, and linked executables are profiled with **`scripts/rocprof-compare-bins.sh`**.

```bash
export CLANG=/path/to/clang    # must support -fopenmp-targets=amdgcn-amd-amdhsa
export LLVM_CONFIG=...
./scripts/build_dataset_amdgpu_loop_unroll.sh
```

If **`rocprof`** or the linker step fails, JSON from **`ll-dump`** is still refreshed; `perf_compare.json` may be sparse until ROCm profiling succeeds on real hardware.

## OpenMP test cases

### OpenMP loop split (`loop-split-openmp/`)

Sources: **`bench_before.c`** (single loop) and **`bench_after.c`** (`#pragma omp split`, OpenMP 6).

Regenerate **`web/datasets/loop-split-openmp/`**:

```bash
export CLANG=/path/to/tiwari_loop_splitting/build/bin/clang
export LLVM_CONFIG=/path/to/build/bin/llvm-config
./scripts/build_dataset_loop_split_openmp.sh
```

`perf_compare.json` is written only when **`perf`** is installed; it is produced by **`scripts/run-perf.sh`**, not edited by hand.
