# Features & Goals

## Goal

Provide an interactive, browser-based visualization of how LLVM loop transformations change a program's control-flow graph, memory access patterns, and performance characteristics — making compiler internals tangible for students, researchers, and compiler engineers.

## Applicability

- **Compiler courses**: Visualize what loop tiling, unrolling, interchange, and fusion actually do to the CFG.
- **Performance debugging**: Correlate a transformation with cache miss rates, IPC changes, and branch mispredictions.
- **Patch review**: Quickly compare before/after CFGs when reviewing LLVM transform patches.
- **Research**: Generate publication-ready visuals of transformation effects.

---

## What This Tool Covers

**Covered — LLVM middle-end loop transforms (IR-level):**
Loop Tiling, Loop Unrolling, Loop Interchange, Loop Fusion, Loop Distribution, Loop Flatten, Loop Unroll-and-Jam, Loop Rotation, LICM, Loop Vectorization (structural effect), and any Polly transform (result on IR). Anything that changes the CFG between two `.ll` files is visible.

**Covered indirectly — OpenMP/Clang AST loop transforms:**
`#pragma omp tile`, `#pragma omp unroll`, `#pragma omp interchange`, `#pragma omp fuse`, `#pragma omp split` restructure the AST before IR generation. The tool shows their *effect on the emitted IR* (compare IR with vs. without the pragma), but cannot visualize the AST transformation itself.

**Not covered:**
- Clang AST-level visualization (use `clang -ast-dump` for that).
- Backend / MachineIR transforms (software pipelining, hardware loops).
- Polly's internal polyhedral schedules and dependence polyhedra (only the resulting IR).
- Source-line correlation (the tool works at IR basic-block granularity).
- Vector type semantics (vectorized IR is shown but not interpreted as SIMD).

## Offline vs. AI-Powered Features

### Fully offline (no AI, no internet)

Everything below is generated by `ll-dump` + `run-perf.sh` and rendered client-side:

| Feature | Source |
|---|---|
| CFG nodes, edges, loop regions | `ll-dump` (LLVM `LoopInfo` + `DominatorTree`) |
| Per-block instruction mix (arith / memory / branch / other) | `ll-dump` (opcode classification) |
| Memory access patterns (stride-1, strided, invariant) | `ll-dump` (LLVM `ScalarEvolution`) |
| Loop-carried, flow, anti, output dependencies | `ll-dump` (LLVM `DependenceInfo`) |
| Trip count, induction variable, loop depth | `ll-dump` (LLVM `ScalarEvolution`) |
| Before/after diff, morph animation | Client-side JS (label comparison) |
| Auto-detection of transform type | Client-side JS (heuristic on CFG structure) |
| Performance bar chart (cycles, IPC, cache, branches) | `run-perf.sh` (Linux `perf stat`, averaged over 5 runs by default; configurable via `PERF_RUNS`) |
| Rule-based insights (improvements, regressions, suggestions) | Client-side JS (static rule engine) |
| Rule-based chat answers | Client-side JS (keyword matching on data) |

### AI-enhanced (requires a model provider)

| Feature | What AI adds |
|---|---|
| Richer "✨ Insights" | Contextual, data-specific explanations — e.g., "The 34% L1 miss rate drop is because tiling keeps the 4×N working set in a 32 KB L1 line." |
| Conversational "💬 Chat" | Free-form Q&A — ask follow-ups, request alternative transforms, compare with other passes. |
| Fallback | If AI is unavailable, both features automatically fall back to rule-based output. |

### How to integrate an AI model (step-by-step)

1. **Click ⚙ AI** in the header bar to open the settings dialog.
2. **Choose a provider** from the dropdown:
   - **Anthropic (Claude)** — best quality. Requires an API key from [console.anthropic.com](https://console.anthropic.com).
   - **OpenAI (GPT-4o)** — requires an API key from [platform.openai.com](https://platform.openai.com).
   - **Local Ollama** — free, fully offline. Install [Ollama](https://ollama.com), run `ollama pull llama3`, start with `OLLAMA_ORIGINS=* ollama serve`.
   - **Custom endpoint** — any OpenAI-compatible API (LM Studio, vLLM, Together AI, etc.).
3. **Enter the model name** (e.g., `claude-sonnet-4-20250514`, `gpt-4o`, `llama3`). Leave blank for the provider's default.
4. **Enter the API key** (Anthropic / OpenAI) or **API endpoint URL** (Ollama / Custom). Keys are stored in browser `localStorage` only.
5. **Click Save**.
6. **Use ✨ Insights → Generate** to get AI-powered analysis, or **💬 Chat** to start a conversation.

If the AI call fails (network error, expired key, model not running), the tool automatically falls back to rule-based insights without user intervention.

## Dataset Dropdown (top-left)

The dropdown serves two purposes:

1. **Reference library** — The built-in datasets (Loop Tiling, Unrolling, Interchange, Fusion, LICM) are curated examples with hand-verified data and static insights. They let new users explore what each transformation looks like without needing LLVM IR files or a build environment — useful for teaching, demos, and quick reference.

2. **Analysis history** — `./scripts/analyze.sh before.ll after.ll` writes `before.json`, `after.json`, and (when `perf` is available) `perf_compare.json` directly under `web/`. To keep multiple analyses selectable from the dropdown, copy those files into `web/datasets/my-project-v1/`, `web/datasets/my-project-v2/`, etc., and add an `<option>` for each folder. Each entry is then a saved snapshot you can switch between without re-running `ll-dump` or `perf`.

To add your own dataset: create `web/datasets/your-name/` with `before.json`, `after.json`, and optionally `perf_compare.json`, then add an `<option value="datasets/your-name">Your Label</option>` to the `<select id="dataset-select">` in `index.html`.

## Feature Reference

### CFG Visualization

| Feature | How to trigger |
|---------|---------------|
| View single CFG | Load `output.json` or a `before.json`; leave **Show diff** off. |
| Side-by-side diff | Click **Show diff**. Nodes are colored: green = new, red = removed, blue = unchanged. |
| Morph animation | Click **Morph** (requires both `before.json` and `after.json`). The before-CFG smoothly animates into the after-CFG. |
| Drag nodes | Click and drag any basic block to rearrange the graph. |
| Zoom / pan | Scroll to zoom, drag background to pan. |
| Hierarchical layout | Click **Hierarchical layout** in the bottom bar for a top-to-bottom Dagre layout. |
| Reset layout | Click **Reset layout** to return nodes to force-directed positions. |
| Dark mode | Click the moon icon (top-right). |

### Node Detail Panel (click any CFG node)

| Feature | What it shows |
|---------|--------------|
| Instruction mix | Stacked color bar — blue (arithmetic), amber (memory), purple (branch), gray (other) — with counts. |
| Memory access pattern | Read/write ratio bar, per-array cards with access type (Sequential/Strided), animated locality cells showing cache-friendliness. |
| Dependencies | Color-coded chips for loop-carried, flow, anti, and output dependencies with distance and description. |
| Loop info | Header, depth, induction variable, trip count. |

### Benchmark Comparison (requires `perf_compare.json`)

| Feature | How to trigger |
|---------|---------------|
| Grouped bar chart | Loads automatically. Shows normalized before/after for execution time, cycles, IPC, L1/LLC miss rates, branch miss rate, instructions. When `perf_compare.json` contains a `"runs"` field, the heading and summary display "averaged over N runs". |
| Absolute values toggle | Click **Show absolute values** above the chart. |
| Speedup summary | Shown automatically — reports percentage speedup and which metric it was computed from. |
| Transformation insights | Scrollable panel below the chart with "What improved", "What regressed", and "Trade-offs" sections. |

### Dataset Switching

| Feature | How to trigger |
|---------|---------------|
| Switch transformation | Use the dropdown at top-left. Ships with curated examples (Loop Tiling, Unrolling, Interchange, Fusion, LICM, Loop Tiling from IR) plus any datasets you add. |
| Add your own dataset | Create `web/datasets/your-transform/` with `before.json`, `after.json`, and optionally `perf_compare.json`. Add an `<option value="datasets/your-transform">` in `index.html`. |

### Auto-Detection

| Feature | How to trigger |
|---------|---------------|
| Transform type detection | Load `before.json` and `after.json` without a `transform` field. The tool infers tiling, unrolling, interchange, fusion, distribution, or rotation from structural differences (loop counts, depths, induction variables, memory patterns). Shown as "Loop Tiling (detected)" in the badge. |

### AI-Powered Insights

| Feature | How to trigger |
|---------|---------------|
| Configure AI provider | Click **⚙ AI** in the header. Choose OpenAI, Anthropic, local Ollama, or a custom endpoint. Save your API key. |
| Generate insights | Click **✨ Insights** → **Generate**. The tool sends the full transformation context (CFG structure, loop info, perf data) to your chosen model and displays categorized results: "What Improved", "What Regressed", "Suggestions", "Applications". |
| AI chatbot | Click **💬 Chat**. A conversational panel opens (bottom-left) pre-loaded with the current transformation data. Ask questions like "Why did L1 miss rate decrease?" or "Would unrolling help here?" |

### Full Pipeline (CLI)

| Feature | How to trigger |
|---------|---------------|
| End-to-end analysis | Run `./scripts/analyze.sh before.ll after.ll`. This compiles both IRs, extracts CFG JSON via `ll-dump`, runs `perf stat` for hardware counters, and opens the visualizer. |
| Perf-only | Run `./scripts/run-perf.sh before.ll after.ll` to generate just `perf_compare.json`. Default 5 runs; override with `PERF_RUNS=10 ./scripts/run-perf.sh …`. |

### Loop Animation (single-view mode)

| Feature | How to trigger |
|---------|---------------|
| Animate loop iteration | Press **Play**. The active basic block highlights and steps through the loop's execution order. |
| Speed control | Drag the **Speed** slider. |
| Choose loop | Use the **Loop** dropdown to pick which loop to animate. |
| Memory access chips | Appear during animation if the loop's JSON includes `memory_accesses`. |
